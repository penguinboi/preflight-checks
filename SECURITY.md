# Security Policy

## Scope

Preflight Checks is a documentation project — guides and checklists, not running
software. That said, security issues can still arise:

- **Inaccurate security guidance** in the playbook that could lead someone to
  ship insecure code
- **Repo-level vulnerabilities** such as malicious link injection in markdown
  files or compromised dependencies in tooling

Both are things we take seriously.

## Reporting a Vulnerability

**Primary:** Use [GitHub private vulnerability reporting][gh-pvr] on this
repository. This keeps the conversation private until a fix is in place.

**Backup:** Email [penguinboisoftware@gmail.com](mailto:penguinboisoftware@gmail.com)
with a description of the issue, affected files, and any suggested fix.

Please do **not** open a public issue for security reports.

[gh-pvr]: https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability

## What to Expect

1. **Acknowledgment** within 72 hours of your report.
2. **Triage** — we'll assess severity and determine the right fix.
3. **Fix and release** — corrections are merged and a new version tagged.
4. **Credit** — reporters are credited in the fix commit and release notes
   unless they prefer to remain anonymous.

## Thank You

If you find something wrong, you're helping every team that relies on these
guides to ship safely. We appreciate it.
